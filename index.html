02 · Evaluation framework

Five dimensions, no automatic composite

The taxonomy is stable across standalone agents, multi-agent systems, and workflows. The assertions underneath it are selected from each fixture's manifest.

Correctness

Required task and environment state; terminal status; cancellation and recovery.

Current: mixed task outcomes; PAL Core not isolated.
G

Groundedness

Source entailment, permitted context, asset/state fidelity, and isolation.

Current: one checked source task and one state sentinel.
Q

Quality

Completeness, coherence, usability, instruction fidelity, and repeat stability.

Current: no calibrated human or model-judge score.
S

Safety

Privacy, permissions, approvals, idempotency, and prohibited side effects.

06 · Graders

Use the cheapest grader that can verify the real outcome

Rule-based

Exact output, schema, fields, list/count, length, sentinels, permissions, terminal state, and backend side effects.

Retain: assertion code/version and every assertion result.

Model-based

Semantic correctness, completeness, citation entailment, plan quality, and workflow coherence.

Retain: frozen judge prompt/model, rationale, evidence span, and uncertainty.

Human / domain expert

Safety-sensitive decisions, high-impact facts, subjective quality, calibration, and disagreements.

Retain: blind rubric verdict, reviewer/calibration ID.

The current sample used deterministic/manual checks and one independent primary-source verification. It did not use a calibrated model judge or blind human panel.

07 · Regression system

Catch PAL regressions without confusing them with agent changes

  1. Freeze the comparison.Version task, fixture, model, tools, assets, permissions, environment, and grader.
  2. Run matched pairs.Compare baseline and candidate on the same task/fixture/environment; never compare unmatched aggregates.
  3. Separate suites.Capability tests ask what can improve; regression tests protect previously passing behavior.
  4. Gate platform risk.Any new critical privacy, approval, idempotency, or prohibited-side-effect violation blocks release.
  5. Use calibrated thresholds.Derive latency, cost, and quality alerts from product SLOs or baseline variability—not universal invented percentages.
  6. Review traces.Inspect every material failure and a rotating sample of passes; quarantine broken tasks.
  7. Learn from incidents.Convert confirmed production failures and grader disputes into minimized permanent cases.
