03 · Task suite architecture

Route arbitrary PAL configurations to the tests that apply

A publishing agent, research agent, multi-agent team, and human-input workflow should not receive one generic benchmark. Every fixture declares a manifest; the evaluator always runs platform-core controls, then selects relevant capability packs and journeys.

Layer 1

Platform core

Universal harness behavior: submission, terminal state, isolation, state, permissions, cancellation, recovery, approval, idempotency, trace and resource accounting.

Layer 2

Capability packs

Tests selected from the fixture manifest: research, transformation, tools, multi-agent, workflow, multilingual, memory and external action.

Layer 3

User journeys

Realistic end-to-end tasks, including small user inputs between workflow steps and downstream state verification.

Layer 4

Incident & holdout

Minimized production failures, adversarial cases and unpublished tasks used to catch regressions and benchmark overfitting.

Suite admission rule

  • Two reviewers can independently apply the pass criteria.
  • A known reference solution passes every blocking grader.
  • The environment and permitted state are reproducible.
  • Positive and negative controls prevent one-sided optimization.
  • Broken, ambiguous or contaminated tasks are quarantined.

04 · Metric contracts

Every number needs a formula, unit and missing-data rule

Metrics are calculated within a declared task family and attribution boundary. Heterogeneous task rows are never pooled into a PAL-wide reliability percentage.

No default composite score. A deployment may introduce weights only after publishing the risk profile, calibration evidence, missing-data treatment and hard gates. Safety failures cannot be averaged away.

08 · Graders

Use the cheapest grader that can verify the real outcome

Rule-based

Exact output, schema, fields, list/count, length, sentinels, permissions, terminal state, and backend side effects.

Retain: assertion code/version and every assertion result.

Model-based

Semantic correctness, completeness, citation entailment, plan quality, and workflow coherence.

Retain: frozen judge prompt/model, rationale, evidence span, and uncertainty.

Human / domain expert

Safety-sensitive decisions, high-impact facts, subjective quality, calibration, and disagreements.

Retain: blind rubric verdict, reviewer/calibration ID.

The current sample used deterministic/manual checks and one independent primary-source verification. It did not use a calibrated model judge or blind human panel.

09 · Regression system

Catch PAL regressions without confusing them with agent changes

  1. Freeze the comparison.Version task, fixture, model, tools, assets, permissions, environment, and grader.
  2. Run matched pairs.Compare baseline and candidate on the same task/fixture/environment; never compare unmatched aggregates.
  3. Separate suites.Capability tests ask what can improve; regression tests protect previously passing behavior.
  4. Gate platform risk.Any new critical privacy, approval, idempotency, or prohibited-side-effect violation blocks release.
  5. Use calibrated thresholds.Derive latency, cost, and quality alerts from product SLOs or baseline variability—not universal invented percentages.
  6. Review traces.Inspect every material failure and a rotating sample of passes; quarantine broken tasks.
  7. Learn from incidents.Convert confirmed production failures and grader disputes into minimized permanent cases.
